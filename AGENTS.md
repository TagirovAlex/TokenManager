# AGENTS.md - Правила для AI-агентов

## Стек технологий
- Python 3.11+, Flask, SQLAlchemy, PostgreSQL
- Debian 12+ без Docker, systemd для управления сервисом
- Минимальные сторонние зависимости

## Архитектура проекта

```
prompt_manager/
├── app/
│   ├── __init__.py      # Flask app factory
│   ├── config.py       # Конфигурация (ВСЕ НАСТРОЙКИ ЗДЕСЬ)
│   ├── models/         # SQLAlchemy модели
│   └── services/       # Бизнес-логика (внутренние вызовы)
├── migrations/         # Alembic
├── tests/             # Тесты
├── requirements.txt
├── run.py            # Точка входа
├── SPEC.md           # Техническое задание
└── TODO.md           # Список задач
```

## Правило 1: Все настройки в config.py

- **Никаких хардкодов** в коде (порты, хосты, пути, размеры)
- Все настройки: через `current_app.config` или `os.getenv`
- Конфигурация хранится ТОЛЬКО в `app/config.py`
- Файл `.env` для локальных переопределений
- Приложение не принимает внешние подключения, все взаимодействия — внутри приложения

## Правило 2: Стандарт именования

### Python (PEP8 + проектные соглашения)

| Что | Стандарт | Пример |
|-----|----------|--------|
| Модули, пакеты | snake_case | `category_service.py` |
| Классы | PascalCase | `CategoryService` |
| Функции | snake_case | `get_categories()` |
| Переменные | snake_case | `category_id` |
| Константы | UPPER_SNAKE_CASE | `MAX_IMAGE_SIZE` |
| Приватные методы | _underline | `_internal_method()` |
| FK поля | `<table>_id` | `category_id`, `object_id` |
| UUID поля | id (строки) | `id = db.Column(db.String(36), PK)` |

### Именование базы данных

| Сущность | Имя таблицы | Пример |
|----------|-------------|--------|
| Модель |Category| `class Category(db.Model)` → `categories` |
| PK | id | UUID строка |
| FK | `<related>_id` | category_id |

## Правило 3: Работа в проекте

- **ЗАПРЕЩЕНО выходить за пределы папки проекта**
- Структура проекта:
  - app/routes/ - веб-роуты (web.py, auth.py)
  - app/services/ - бизнес-логика (7 сервисов)
  - app/models/ - модели БД
  - app/templates/ - HTML шаблоны
  - app/static/ - CSS, JS

## Правило 6: Оптимизация работы

- НЕ читать TODO.md - он завершен
- Использовать конкретные пути в grep/glob вместо глобальных поисков
- Читать только нужные участки файлов (offset/limit)
- Минимизировать параллельные запросы - только для независимых задач
- Спрашивать перед поиском по всему проекту

## Правило 4: Удаление файлов

- **Удалять файлы только в папку `.trash`**
- Папка `.trash` должна быть в .gitignore
- Реальное удаление из .trash — только вручную пользователем

## Правило 5: Этапы разработки

**Текущий этап: Написание кода**
- Писать код без запуска тестов и развертывания
- Проверять синтаксис и ошибки вручную
- Тесты и развертывание — в отдельном этапе
- **Файлы проекта и развертывания (install.sh, uninstall.sh) проверять только при создании нового тега/релиза**

## Ключевые соглашения

### Всегда использовать
- UUID для всех PK (строка 36 символов)
- SQLAlchemy ORM (не raw SQL)
- Alembic для миграций БД
- Конфигурация через переменные окружения

### Категории и объекты
- Новые категории добавляются через админку (динамическая система)
- Каждый объект принадлежит одной категории
- У категории могут быть динамические атрибуты

### Атрибуты (AttributeDef)
- Типы полей: `bool`, `int`, `int_list`, `str`, `str_list`
- `int_list`: min, max, step
- `str_list`: JSON массив опций

### Шаблоны промтов
- template_text: "{obj_id} идет в {loc_id}"
- Генератор заменяет {id} на prompt объекта
- Результат сохраняется в TemplateResult

## Процедура коммита

1. **git status** - проверить изменения
2. **git diff** - посмотреть что изменилось
3. **git add -A** - за stage все изменения
4. **git commit -m "сообщение"** - закоммитить
5. **git push** - запушить

**Сообщение коммита** - краткое (1-2 предложения), что сделано
